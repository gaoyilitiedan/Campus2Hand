## 1. 用户故事

### 1.1 学生买家（Buyer）

| 编号    | 用户故事                                                                 |
|---------|--------------------------------------------------------------------------|
| US-B01  | 作为学生买家，我希望能够按关键词、分类、价格区间、新旧程度搜索商品，以便快速找到心仪的二手教材或闲置物品。 |
| US-B02  | 作为学生买家，我希望能够查看商品详情（图片、描述、价格、卖家信誉分、所在校区），以便判断商品是否值得购买。 |
| US-B03  | 作为学生买家，我希望能够与卖家在线聊天沟通，以便确认商品成色、交易地点等细节后再下单。 |
| US-B04  | 作为学生买家，我希望能够通过微信支付在线付款，资金先进入平台担保，以便确认收货满意后卖家才能收到钱。 |
| US-B05  | 作为学生买家，我希望能够在收到商品满意后确认收货，不满意时申请退款并填写原因，以便保障自己的消费权益。 |
| US-B06  | 作为学生买家，我希望能够在交易完成后给卖家打分和评价，以便为其他同学提供参考并共建诚信校园交易环境。 |
| US-B07  | 作为学生买家，我希望能够查看卖家的信誉分和历史评价，以便选择靠谱的交易对象。 |

### 1.2 学生卖家（Seller）

| 编号    | 用户故事                                                                 |
|---------|--------------------------------------------------------------------------|
| US-S01  | 作为学生卖家，我希望能够发布闲置商品（上传图片、填写标题/描述/价格/分类/新旧程度），以便让全校同学看到并购买。 |
| US-S02  | 作为学生卖家，我希望能够管理自己在售的商品列表（编辑、下架、标记已售出），以便灵活调整出售计划。 |
| US-S03  | 作为学生卖家，我希望能够通过平台聊天与买家沟通交易细节（如校内面交地点），以便快速促成交易。 |
| US-S04  | 作为学生卖家，我希望能够在买家确认收货后收到货款，金额自动结算到我的微信零钱，以便安全便捷地变现闲置。 |
| US-S05  | 作为学生卖家，我希望能够在交易完成后给买家打分和评价，以便筛选有诚信的买家并积累自己的平台信誉。 |
| US-S06  | 作为学生卖家，我希望能够看到自己的信誉分和累计成交记录，以便长期建立可靠卖家身份。 |

### 1.3 通用用户（Common）

| 编号    | 用户故事                                                                 |
|---------|--------------------------------------------------------------------------|
| US-C01  | 作为学生用户，我希望通过学号和校园卡密码完成实名认证，以便获得平台完整功能权限（发布、购买、聊天）。 |
| US-C02  | 作为学生用户，我希望在忘记密码时能通过校园邮箱重置密码，以便快速恢复账户访问。 |
| US-C03  | 作为学生用户，我希望个人信息中的学号仅脱敏展示，手机号、宿舍地址等隐私信息不在商品页面暴露，以便保护个人隐私。 |
| US-C04  | 作为学生用户，我希望在遇到交易纠纷时能够提交投诉工单，由平台介入仲裁处理，以便维护自身合法权益。 |

## 2. 功能列表

### 2.1 用户认证模块（US-C01、US-C02）

#### F-01 注册与实名认证

| 步骤 | 功能描述 |
|------|----------|
| F-01-1 | 用户输入学号，系统校验学号格式（长度、字符集） |
| F-01-2 | 用户输入校园卡密码，系统调用校园系统接口校验身份真实性 |
| F-01-3 | 校验通过后，用户设置平台登录密码（长度 ≥ 8 位，含大小写字母 + 数字） |
| F-01-4 | 用户绑定校园邮箱（格式校验 @campus.edu 域名校验） |
| F-01-5 | 系统发送邮箱验证链接到用户校园邮箱 |
| F-01-6 | 用户点击邮箱验证链接完成邮箱绑定 |
| F-01-7 | 实名认证完成后，用户获得完整功能权限（发布、购买、聊天） |
| F-01-8 | 用户可在注册后绑定手机号（可选），用于安全通知和账号找回 |

#### F-02 登录

| 步骤 | 功能描述 |
|------|----------|
| F-02-1 | 用户输入学号 + 密码登录 |
| F-02-2 | 连续 5 次失败后，账户锁定 30 分钟 |
| F-02-3 | 登录成功后返回 Access Token（有效期 2h）+ Refresh Token（有效期 7d） |
| F-02-4 | Access Token 过期后用 Refresh Token 换取新 Token，无需重新登录 |
| F-02-5 | 支持微信小程序一键授权登录（已绑定学号的前提下） |

#### F-03 忘记密码 / 重置密码

| 步骤 | 功能描述 |
|------|----------|
| F-03-1 | 用户输入学号，点击"忘记密码" |
| F-03-2 | 系统向该学号绑定的校园邮箱发送密码重置链接 |
| F-03-3 | 用户点击链接，进入重置密码页面 |
| F-03-4 | 输入新密码（长度 ≥ 8 位，含大小写字母 + 数字）并确认 |
| F-03-5 | 重置成功后，所有已登录设备强制登出 |
| F-03-6 | 已知旧密码时，用户可在个人设置中直接修改密码 |

#### F-04 商品搜索

| F-04-5 | 搜索结果支持排序（按发布时间、价格升序/降序、信誉分降序） |

#### F-05 商品详情

|------|----------|

| F-05-2 | 展示商品标题、价格、新旧程度、分类标签 |
| F-05-3 | 展示商品详细文字描述 |
| F-05-4 | 展示卖家脱敏信息：昵称、信誉分（⭐ 1-5 分）、所在校区 |
| F-05-5 | 展示"联系卖家"按钮，点击跳转聊天页面 |
| F-05-6 | 展示"立即购买"按钮，点击跳转支付流程 |
| F-05-7 | 展示卖家历史评价列表（最近 20 条，支持查看更多） |

### 2.3 商品发布与管理模块（US-S01、US-S02）

#### F-06 发布商品

| 步骤 | 功能描述 |
|------|----------|
| F-06-1 | 用户上传商品图片（至少 1 张，最多 9 张，支持 JPG/PNG/WebP，单张 ≤ 10MB） |
| F-06-2 | 图片上传后系统自动压缩生成缩略图，原图存储至 OSS |
| F-06-3 | 用户填写商品标题（1-60 字符，必填） |
| F-06-4 | 用户填写商品描述（1-2000 字符，必填） |
| F-06-5 | 用户设置商品售价（0.01-99999.99 元，必填） |
| F-06-6 | 用户选择商品分类（必选） |
| F-06-7 | 用户选择新旧程度（必选） |
| F-06-8 | 用户选择所在校区（从已认证信息自动拉取，可手动修改） |
| F-06-9 | 提交后系统进行内容审核（敏感词过滤、违规图片检测），审核通过后发布上线 |
| F-06-10 | 审核未通过时返回具体原因，用户可修改后重新提交 |

#### F-07 管理商品

| 步骤 | 功能描述 |
|------|----------|
| F-07-1 | 查看"我的在售"商品列表，分状态 Tab（在售 / 审核中 / 已售出 / 已下架） |
| F-07-2 | 编辑商品信息（仅限在售和审核中状态），修改后重新进入审核 |
| F-07-3 | 下架商品（从公开列表移除，保留在个人列表） |
| F-07-4 | 标记为已售出（手动标记，用于面交后没有走平台支付的场景） |
| F-07-5 | 删除商品（仅限已下架状态，软删除） |

### 2.5 支付与交易模块（US-B04、US-B05、US-S04）

#### F-09 下单与支付

| 步骤 | 功能描述 |
|------|----------|
| F-09-1 | 用户在商品详情页点击"立即购买"，进入订单确认页 |
| F-09-2 | 系统展示订单摘要（商品信息、金额、卖家昵称） |
| F-09-3 | 用户确认后，系统创建订单（状态：待付款），并生成支付单 |
| F-09-4 | 系统调用微信支付统一下单接口，返回支付参数 |
| F-09-5 | 用户完成微信支付后，微信回调通知平台 |
| F-09-6 | 平台验证回调签名后，更新订单状态为"已付款（担保中）" |
| F-09-7 | 资金进入平台担保账户，订单状态变更为"待发货" |
| F-09-8 | 订单创建后 30 分钟未支付，自动取消 |

#### F-10 确认收货

| 步骤 | 功能描述 |
|------|----------|
| F-10-1 | 面交完成后，买家在订单详情页点击"确认收货" |
| F-10-2 | 系统弹出二次确认弹窗："确认已收到商品？确认后资金将打款给卖家" |
| F-10-3 | 买家确认后，订单状态变更为"已完成"；若结算失败则变为"结算异常"，后台重试 |
| F-10-4 | 系统触发结算：平台担保金自动结算至卖家微信零钱 |
| F-10-5 | 双方可互相评价 |

#### F-11 申请退款

| 步骤 | 功能描述 |
|------|----------|
| F-11-1 | 买家在"已付款（担保中）"或"待发货"状态下可申请退款 |
| F-11-2 | 买家选择退款原因（商品与描述不符 / 卖家无法联系 / 商品有损坏 / 其他） |
| F-11-3 | 买家填写退款说明（1-500 字符，必填） |
| F-11-4 | 支持上传举证图片（最多 3 张，单张 ≤ 5MB） |
| F-11-5 | 提交后订单状态变更为"退款中" |
| F-11-6 | 系统通知卖家，卖家可在 48 小时内同意退款或拒绝 |
| F-11-7 | 卖家同意 → 退款原路退回买家微信，订单状态变更为"已退款" |
| F-11-8 | 卖家拒绝 / 48 小时超时未处理 → 自动升级为纠纷工单，平台介入仲裁 |

#### 订单状态机（完整流转图）

```
                    ┌──────────────┐
                    │ pending_payment│ (待付款)
                    └───────┬──────┘
                            │ 支付成功
                    ┌───────▼──────┐
                    │  paid_escrow  │ (已付款-担保中)
                    └───────┬──────┘
                            │ 进入担保
                    ┌───────▼──────────┐
                    │ pending_shipment  │ (待发货)
                    └───────┬──────────┘
                            │
              ┌─────────────┼──────────────┐
              │ 买家确认收货  │              │ 买家申请退款
     ┌────────▼──────┐      │     ┌────────▼────────┐
     │   completed   │      │     │    refunding    │ (退款中)
     │   (已完成)     │      │     └────────┬────────┘
     └────────┬──────┘      │              │
              │结算失败       │     ┌────────┼────────┐
     ┌────────▼──────────┐  │     │同意     │拒绝     │超时48h
     │settlement_failed  │  │ ┌───▼──┐ ┌──▼─────┐┌──▼──────┐
     │  (结算异常-重试)   │  │ │refunded│ │paid_   ││自动升级 │
     └───────────────────┘  │ │(已退款)│ │escrow  ││纠纷工单 │
                            │ └───────┘ │(恢复)  │└─────────┘
                            │           └────────┘
                    
     ┌────────▼──────────┐
     │   cancelled       │ (已取消 - 30min未支付)
     └───────────────────┘
```

**终态：** `completed`、`refunded`、`cancelled`
**非终态：** `pending_payment`、`paid_escrow`、`pending_shipment`、`refunding`、`settlement_failed`

### 2.6 评价与信誉系统（US-B06、US-B07、US-S05、US-S06）

#### F-12 交易评价

| 步骤 | 功能描述 |
|------|----------|
| F-12-1 | 订单完成后，双方均可进入评价页面 |
| F-12-2 | 用户给出 ⭐ 评分（1-5 分，整数） |
| F-12-3 | 用户填写文字评价（1-500 字符，可选） |
| F-12-4 | 用户可选评价标签（如"沟通顺畅""商品如实""发货快"等预置标签） |
| F-12-5 | 评价提交后不可修改（保证评价真实性） |
| F-12-6 | 评价在双方均提交后公开可见；单方评价 7 天后自动公开 |

#### F-13 信誉分计算

| 步骤 | 功能描述 |
|------|----------|
| F-13-1 | 信誉分 = 所有已生效评价的平均星数（保留 1 位小数） |
| F-13-2 | 无评价用户默认信誉分 = 3.0（中等信誉） |
| F-13-3 | 信誉分在每次新评价生效后实时更新 |
| F-13-4 | 用户个人主页展示信誉分 + 评价总数 + 成交总数 |

### 2.7 纠纷与投诉模块（US-C04）

#### F-14 投诉工单

| 步骤 | 功能描述 |
|------|----------|
| F-14-1 | 用户在订单详情页点击"申请平台介入"创建投诉工单 |

| F-14-7 | 双方可在工单中追加留言和凭证 |
| F-14-8 | 工单解决后，双方收到处理结果通知 |

### 2.8 隐私保护模块（US-C03）

#### F-15 隐私脱敏

| 步骤 | 功能描述 |
|------|----------|
| F-15-1 | 学号在所有前端页面仅展示前 2 位 + 后 2 位，中间以 **** 代替（如 20****01） |
| F-15-2 | 手机号在任何页面均不对外展示，仅用于系统通知 |
| F-15-3 | 宿舍地址 / 详细地址字段仅在用户个人设置中可见，不暴露于商品页、聊天页、订单页 |
| F-15-4 | 聊天中如需约定面交地点，由用户自行输入，系统不做限制 |
| F-15-5 | API 返回数据中，敏感字段由后端统一脱敏后再返回前端 |

## 3. 验收标准

### 3.1 用户认证模块

| 编号 | 需求描述 | 技术方案 |
|------|----------|----------|
| NF-S01 | 传输层加密 | 全站 **HTTPS**（TLS 1.2+），禁止 HTTP 明文传输 |
| NF-S02 | 密码存储加密 | 使用 **bcrypt**（cost≥10）单向哈希存储，禁止明文或可逆加密 |
| NF-S03 | JWT Token 安全 | Token 使用 **HS256** 签名，含过期时间（2h），敏感接口校验 Token 有效性 |
| NF-S04 | API 防重放攻击 | 关键接口（支付、退款）附带 **nonce + timestamp**，nonce 5 分钟内不可重复使用 |
| NF-S05 | SQL 注入防护 | 所有数据库操作使用 **参数化查询**（ORM），禁止拼接 SQL |
| NF-S06 | XSS 攻击防护 | 所有用户输入输出做 **HTML 实体编码** 转义，CSP 头配置 |
| NF-S07 | CSRF 防护 | 使用 **CSRF Token**（SPA 应用使用 SameSite Cookie + Origin 头校验） |
| NF-S08 | 文件上传安全 | 校验文件 **MIME 类型 + 文件头魔数**，禁止可执行文件上传；上传目录禁止脚本执行 |
| NF-S09 | 微信支付签名 | 所有请求/回调使用 **HMAC-SHA256** 签名，商户 Key 加密存储于环境变量 |
| NF-S10 | 敏感信息日志脱敏 | 日志中禁止打印密码、Token、手机号、完整学号，均已 **** 替代 |
| NF-S11 | 账号安全 | 异地登录检测 + 通知；支持查看最近 **10 次** 登录记录 |

## 6. API 接口设计

### 6.1 通用约定

| 项目 | 约定 |
|------|------|
| 基础路径 | `/api/v1` |
| 协议 | HTTPS |
| 数据格式 | JSON（请求/响应），Content-Type: `application/json` |
| 字符编码 | UTF-8 |
| 认证方式 | Bearer Token（Header: `Authorization: Bearer <JWT>`） |
| 通用响应格式 | `{ "code": 0, "message": "success", "data": {...} }` |
| 分页格式 | `{ "page": 1, "size": 20, "total": 150, "list": [...] }` |
| 时间格式 | ISO 8601 UTC（`2026-05-13T08:00:00Z`） |
| 金额单位 | **分**（避免浮点精度问题），前端展示时转换为元 |

#### 通用错误码

| code | 说明 |
|------|------|
| 0    | 成功 |
| 1001 | 参数校验失败 |
| 1002 | 资源不存在 |
| 1003 | 无权限 |
| 1004 | Token 过期或无效 |
| 1005 | 请求过于频繁 |
| 2001 | 用户未实名认证 |
| 2002 | 账户已锁定 |
| 3001 | 支付失败 |
| 3002 | 订单状态不合法 |
| 5000 | 服务器内部错误 |

### 6.9 微信支付回调（服务端接收）

```
POST /api/v1/callback/wechat-pay
```

此接口由微信支付平台回调，不含 JWT 认证，使用微信签名验签。请求/响应格式遵循 [微信支付 V3 API 回调规范](https://pay.weixin.qq.com/docs/merchant/apis/combine-payment/notification.html)。

**验签流程：**
1. 获取 HTTP 头 `Wechatpay-Signature` 中的签名
2. 使用微信平台公钥验证签名
3. 验签通过后解密回调报文（AEAD_AES_256_GCM）
4. 更新订单状态并返回 HTTP 200 + `{ "code": "SUCCESS" }`

## 7. 数据库核心表设计概要

### 7.1 表清单

| 表名 | 说明 |
|------|------|
| `users` | 用户表（学号、密码哈希、邮箱、昵称、校区、信誉分等） |
| `products` | 商品表（标题、描述、价格、图片JSON、分类、新旧程度、状态等） |
| `orders` | 订单表（订单号、买家ID、卖家ID、商品ID、金额、状态、过期时间等） |
| `conversations` | 会话表（买家ID、卖家ID、商品ID、最后消息时间等） |
| `messages` | 消息表（会话ID、发送者ID、消息类型、内容、状态等） |
| `reviews` | 评价表（订单ID、评价人ID、被评人ID、评分、内容、标签JSON等） |
| `disputes` | 投诉工单表（订单ID、创建人ID、类型、描述、状态、处理人等） |
| `dispute_comments` | 工单留言表（工单ID、发送者ID、内容等） |
| `payments` | 支付记录表（订单ID、微信交易号、金额、状态等） |

### 7.2 关键索引建议

| 表 | 索引字段 | 类型 |
|------|----------|------|
| `products` | `(status, category, price, created_at)` | 联合索引 |
| `products` | `title, description` | 全文索引 (FULLTEXT) |

| `orders` | `seller_id, status` | 联合索引 |

| `reviews` | `reviewee_id, created_at` | 联合索引 |
| `disputes` | `order_id, status` | 联合索引 |
| `users` | `student_id` | 唯一索引 |
